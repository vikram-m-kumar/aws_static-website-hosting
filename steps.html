How to Host a Static Website on AWS

Home Steps
  1. Create an S3 Bucket:
    - Go to the AWS Console → S3 → Create bucket
    - Name the bucket exactly like your domain (e.g., example.com)
    - Uncheck "Block all public access" (you will manage access manually)
    - Enable static website hosting under the "Properties" tab
  2. Upload Website Files:
    - Upload your HTML, CSS, and image files (e.g., index.html)
    - Set index.html as the default root object in static website hosting settings
  3. Make Files Public:
    - Set a bucket policy to allow public read access
    - Sample policy: 
    {
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PublicReadGetObject",
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::example.com/*"
  }]
}
  4. Request SSL Certificate (HTTPS):
    - Go to AWS Certificate Manager (ACM) → Request certificate
    - Use your domain (e.g., example.com) and subdomain (www.example.com)
    - Validate via DNS (copy DNS records into Route 53)
  5. Set Up CloudFront:
    - Create a CloudFront distribution
    - Use your S3 static website endpoint as the origin
    - Attach the SSL certificate from ACM
    - Enable redirect HTTP to HTTPS
  6. Register or Transfer Domain:
    - Use Route 53 to register a new domain or transfer an existing one
  7. Create a Hosted Zone:
    - In Route 53 → Hosted Zones → Create hosted zone
    - Add an A record with **Alias** pointing to the CloudFront distribution
  8. Test Your Website:
    - Open your domain in a browser (e.g., https://www.example.com)
    - You should see your website live and served over HTTPS
